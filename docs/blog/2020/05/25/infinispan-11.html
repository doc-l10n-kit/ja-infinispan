Infinispan 11.0.0.CR1

May 25, 2020 Tags: release

By Tristan Tarrant

親愛なるInfinispanのコミュニティへ

11.0.0.CR1をリリースしたばかりですが、これにはいくつかの機能や改善点が搭載されています。

拠点横断レプリケーションの改善

拠点横断レプリケーション機能には、2つの大きな改善点がありました。

  • 複数の拠点マスターをサポート

    新しいアルゴリズムでは、複数の拠点マスターを活用し、それらの間で負荷を分散できます。環境が可能であれば、RELAY2の構成で拠点マスターの数を増やしてください。

    <relay.RELAY2 site="LON" max_site_masters="1000"/>

  • 非同期拠点横断レプリケーションにおける衝突の検出と解決

    ベクトルクロックを利用して、非同期モードでは、Infinispanは衝突を検出できます。衝突は、2つ以上の拠点が同時に同じキーを更新した場合に発生します。競合は拠点名に基づいて解決されます。競合が発生した場合は、辞書的に下位のサイトからの更新が優先されます。例えば、LON（ロンドン）, NYC（ニューヨーク）, SFO（サンフランシスコ）の3つの拠点がある場合、LONとNYCが競合した場合、NYCからの更新は破棄され、LONからの更新は保持されます。拠点名の前に数字を付加することで、優先度を選択できます。例えば、NYCからの更新を優先したい場合は、拠点名を「1-NYC」に設定します。

将来のアップデートでは、Infinispanでカスタムの競合解決アルゴリズムを設定できるようになります。

ノンブロッキングの内部

  • すべてのスレッドプールは、ブロッキングとノンブロッキングの2つのプールと化しました。

永続性

  • 新しいノンブロッキングストアSPI

クラスタリング

  • より正確なセグメントリバランス

  • 状態遷移を伴わないスケーリング(実験)

問い合せ/索引

  • SearchManager、CacheQuery は非推奨です。ユーザは、Search.getQueryFactory から取得した Ickle String 問い合せに移行することをお勧めします。

サーバーのセキュリティ

  • サーバがデフォルトで安全になりました

  • 新しい暗号化されたプロパティのレルム

  • 認証と認可の設定を簡素化し、可能な限り自動化を実現

その他サーバーの変更

  • データソース

  • ランタイムロギングの設定

  • サーバーレポート

CLI

  • ユーザツールをCLIに統合

  • CLIをネイティブでコンパイルできる

  • 非対話式な使用法

コンソール

  • 多くのUX改善

Kubernetes/OpenShiftオペレータ

  • キャッシュ CR

  • IngressとRouteを使ったサービスの公開

コンテナイメージ

  • ubi-minimal:8.2に基づくイメージ

  • すべてのイメージで設定ファイルをネイティブに生成

  • ネイティブサーバイメージは infinispan/server-native で利用可能 (実験)

ドキュメント

  • RemoteCacheConfigurationBuilder を使用した、ローリング・アップグレードの手続き的なコンテンツ、オペレータとの Cache CR、サーバのパッチ適用、その他 CLI コマンドが追加されました。

  • アップグレードガイドに含まれるさまざまなアップグレードおよび移行作業のための手続き内容です。

  • オペレータとSpring Boot Starterガイドでは、索引ページから安定版と開発版を提供するようになりました。

  • index.htmlとドキュメント全体を更新し、高レベルのコンテキストと検索性を向上させました。

  • 入門のコンテンツが更新され、合理化されました。

  • コミュニティのフィードバックを介して、いくつかの変更、追加、削除をドキュメントに適用しました。

手に入れて、使って、尋ねて！
